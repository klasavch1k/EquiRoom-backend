# API: Критерии оценивания дивизионов

## Общее описание
Критерии привязаны к **дивизиону** и распространяются на **все классы всех секций** этого дивизиона.
Организатор может составлять/редактировать критерии **до старта шоу** (`show.startDate`).
Обычный пользователь и судья могут только просматривать.

---

## 1. Получение критериев

```
GET /api/v1/divisions/{divisionId}/criteria
```

**Авторизация:** Bearer-токен (любой аутентифицированный пользователь).

**Response 200:**
```json
{
  "divisionId": 42,
  "showId": 7,
  "canEdit": true,
  "lockedReason": null,
  "updatedAt": "2026-04-07T12:00:00",
  "version": 3,
  "items": [
    { "id": 101, "name": "Качество покраски", "maxScore": 10, "position": 1 },
    { "id": 102, "name": "Анатомия",          "maxScore": 15, "position": 2 },
    { "id": 103, "name": "Общее впечатление", "maxScore": 5,  "position": 3 }
  ]
}
```

| Поле           | Тип               | Описание                                                                 |
|----------------|--------------------|--------------------------------------------------------------------------|
| `divisionId`   | Long               | ID дивизиона                                                             |
| `showId`       | Long               | ID шоу (для навигации / логики на фронте)                                |
| `canEdit`      | boolean            | `true` = текущий пользователь — организатор И шоу не стартовало          |
| `lockedReason` | String \| null     | Причина блокировки: `"Шоу уже началось"`, `"Шоу завершено"`, или `null`  |
| `updatedAt`    | ISO DateTime       | Время последнего обновления шоу                                          |
| `version`      | Long               | Версия дивизиона (для PUT — оптимистичная блокировка)                    |
| `items`        | CriterionItemDto[] | Критерии, отсортированные по `position`                                  |

**`CriterionItemDto`:**

| Поле       | Тип    | Описание                                 |
|------------|--------|------------------------------------------|
| `id`       | Long   | ID критерия (null при создании нового)    |
| `name`     | String | Название критерия                         |
| `maxScore` | Integer| Максимальный балл (≥ 1)                  |
| `position` | Integer| Порядковый номер (1-based, непрерывный)   |

**Коды ответа:**
- `200` — OK
- `404` — дивизион не найден

## 2. Сохранение критериев (bulk save)

```
PUT /api/v1/divisions/{divisionId}/criteria
```

**Авторизация:** Bearer-токен (только организатор, до старта шоу).

**Request Body:**
```json
{
  "version": 3,
  "items": [
    { "name": "Качество покраски",  "maxScore": 10, "position": 1 },
    { "name": "Анатомия",           "maxScore": 15, "position": 2 },
    { "name": "Общее впечатление",  "maxScore": 5,  "position": 3 },
    { "name": "Новый критерий",     "maxScore": 8,  "position": 4 }
  ]
}
```

| Поле      | Тип                | Обязательно | Описание                                              |
|-----------|--------------------|-------------|-------------------------------------------------------|
| `version` | Long               | ✅          | Версия из последнего GET (оптимистичная блокировка)   |
| `items`   | CriterionItemDto[] | ✅ (≥ 1)   | Полный список критериев в итоговом порядке             |

**Правила валидации:**
- `name` — не пустой, не пробелы
- `maxScore` — ≥ 1
- `position` — ≥ 1, **уникальный**, **непрерывный** (1, 2, 3, ..., N)
- Минимум 1 критерий в списке

**Response 200:** Такой же `CriteriaResponse` с обновлёнными данными и новой `version`.

**Коды ответа:**
- `200` — OK
- `400` — невалидные данные (позиции не непрерывны, пустое имя, и т.д.)
- `403` — нет прав (пользователь не организатор)
- `404` — дивизион не найден
- `409` — конфликт версий (данные были изменены другим пользователем) ИЛИ шоу уже стартовало
- `422` — ошибки Jakarta Validation (@NotBlank, @Min, и т.д.)

## 3. Копирование критериев из другого дивизиона

```
POST /api/v1/divisions/{divisionId}/criteria/copy
```

**Авторизация:** Bearer-токен (только организатор, до старта шоу).

**Request Body:**
```json
{
  "sourceDivisionId": 43,
  "replace": true
}
```

| Поле              | Тип     | Обязательно | Описание                                          |
|-------------------|---------|-------------|---------------------------------------------------|
| `sourceDivisionId`| Long    | ✅          | ID дивизиона-источника (из того же шоу)           |
| `replace`         | boolean | ❌          | `true` = заменить существующие критерии целиком    |

**Проверки на сервере:**
- Оба дивизиона принадлежат одному шоу
- Целевой дивизион доступен для редактирования (организатор + не стартовал)
- Исходный дивизион содержит хотя бы 1 критерий

**Response 200:** Такой же `CriteriaResponse` для целевого дивизиона (с новыми скопированными данными).

**Коды ответа:**
- `200` — OK
- `400` — дивизионы в разных шоу / у источника нет критериев
- `403` — нет прав
- `404` — дивизион не найден
- `409` — шоу уже стартовало / завершено

## Логика фронта

### Открытие модалки критериев
1. На каждом дивизионе — кнопка со значком таблицы (📊).
2. По клику: `GET /api/v1/divisions/{divisionId}/criteria`
3. Если `canEdit === false` → показать таблицу **только для чтения**.
4. Если `canEdit === true` → показать редактируемую таблицу + кнопки.

### Редактирование (организатор, canEdit === true)
- Таблица из двух столбцов: **Название критерия** | **Макс. балл**
- Кнопка **+ Добавить критерий** (добавляет строку в конец)
- Кнопка **🗑 Удалить** у каждой строки (минимум 1 строка должна остаться)
- Drag & drop для перестановки строк (position пересчитывается на фронте 1..N)
- Кнопка **💾 Сохранить** → `PUT /api/v1/divisions/{divisionId}/criteria`
  - Отправить `version` из последнего GET
  - Если 409 → показать «Данные были изменены. Обновите страницу.»

### Копирование
- Кнопка **📋 Дублировать из...** + выпадающий список других дивизионов текущего шоу
- Если таблица **уже содержит данные** → confirm: «Вы уверены? Текущие критерии будут заменены.»
- После подтверждения → `POST /api/v1/divisions/{divisionId}/criteria/copy`
  - `{ "sourceDivisionId": <выбранный>, "replace": true }`
- Ответ сразу содержит новую таблицу → обновить UI

### Просмотр (обычный пользователь)
- Та же модалка, но без кнопок редактирования
- `canEdit === false` → readonly

## Модели данных (БД)

### Новая таблица: `division_criteria`

| Колонка       | Тип                | Описание                      |
|---------------|---------------------|-------------------------------|
| `id`          | BIGSERIAL (PK)      | Автоинкремент                 |
| `division_id` | BIGINT FK → divisions| Дивизион                     |
| `name`        | VARCHAR NOT NULL     | Название критерия             |
| `max_score`   | INT NOT NULL         | Максимальный балл (≥ 1)      |
| `position`    | INT NOT NULL         | Порядок (1-based, уникален в рамках дивизиона) |
| `created_at`  | TIMESTAMP            | Дата создания                 |
| `updated_at`  | TIMESTAMP            | Дата последнего изменения     |

**UNIQUE(division_id, position)**

### Изменение в таблице `divisions`

Добавлена колонка `version` (BIGINT) для оптимистичной блокировки через JPA `@Version`.
